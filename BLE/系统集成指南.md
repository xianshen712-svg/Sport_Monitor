# Sport_Monitor BLE数据到后端服务器集成指南

## 已完成的工作

### ✅ 1. MQTT数据发布模块
**文件**: `BLE/mqtt_publisher.py`
- 通用的MQTT数据发布器，支持多种健康数据类型
- 包含连接管理、错误处理和自动重连功能
- 支持MQTT v3.1.1协议，兼容现有的MQTT服务器

### ✅ 2. 心率采集脚本集成MQTT
**文件**: `BLE/fixed_heart_rate_tool.py`
- 添加了MQTT初始化函数 `init_mqtt()` 和 `cleanup_mqtt()`
- 修改了 `save_heart_rate()` 函数，在保存数据的同时通过MQTT发送到后端服务器
- 保持了原有的本地文件保存功能作为备份

### ✅ 3. 统一的BLE数据采集工具
**文件**: `BLE/unified_ble_mqtt_tool.py`
- 整合了心率、血氧、步数、体温四种数据类型的解析
- 使用统一的MQTT发布器将所有数据实时发送到后端服务器
- 支持同时监听多种健康数据，提高效率

### ✅ 4. 测试脚本
**文件**: 
- `BLE/test_mqtt_integration.py` - 集成测试脚本
- `BLE/simple_mqtt_test.py` - 简单的MQTT连接测试

## 系统架构

```
KCT手表 (CC30设备)
     ↓ (BLE蓝牙)
BLEDebug.exe (Windows工具)
     ↓ (BLE连接)
Python采集脚本 (unified_ble_mqtt_tool.py)
     ↓ (MQTT协议)
MQTT服务器 (localhost:1883)
     ↓ (MQTT订阅)
后端服务器 (Node.js + Express)
     ↓ (WebSocket)
前端界面 (Vue.js + ECharts)
     ↓ (MySQL存储)
数据库 (健康数据持久化)
```

## 快速启动指南

### 步骤1：安装必要的Python包
```bash
cd Sport_Monitor\BLE
pip install paho-mqtt bleak
```

### 步骤2：启动系统服务（按顺序）

#### 终端1：启动MQTT服务器
```bash
cd Sport_Monitor
node simple_mqtt_broker.js
```
**预期输出**:
```
MQTT代理服务器运行在端口 1883
等待客户端连接...
```

#### 终端2：启动后端服务器
```bash
cd Sport_Monitor\server
npm start
```

#### 终端3：启动前端服务器
```bash
cd Sport_Monitor\client
npm run dev
```

#### 终端4：测试MQTT连接
```bash
cd Sport_Monitor\BLE
python simple_mqtt_test.py
```

#### 终端5：运行BLE数据采集
```bash
cd Sport_Monitor\BLE
python unified_ble_mqtt_tool.py
```

## 故障排除

### 问题1：MQTT连接失败
**症状**: `❌ MQTT连接失败` 或 `Invalid header flag bits` 错误

**解决方案**:
1. 确保MQTT服务器正在运行
2. 检查端口1883是否被占用
3. 重启MQTT服务器
4. 使用 `python simple_mqtt_test.py` 测试基本连接

### 问题2：BLE设备连接失败
**症状**: `❌ 未找到目标设备`

**解决方案**:
1. 运行 `BLEDebug.exe` 并连接手表
2. 确保手表蓝牙已开启
3. 检查设备MAC地址是否正确
4. 重启手表蓝牙

### 问题3：数据不显示在前端
**症状**: 前端页面显示模拟数据而非真实数据

**解决方案**:
1. 检查后端服务器是否正在运行
2. 查看WebSocket连接状态
3. 验证数据格式是否符合后端期望
4. 检查数据库连接配置

## 技术支持

### 常见问题解答
**Q: 为什么数据没有实时更新？**
A: 检查MQTT服务器、后端服务器和前端服务器的连接状态，确保所有服务都在运行。

**Q: 如何修改设备MAC地址？**
A: 在 `BLE/unified_ble_mqtt_tool.py` 中修改 `TARGET_MAC` 变量。

**Q: 如何查看详细的调试信息？**
A: 在MQTT发布器中启用调试模式，或查看各服务的日志文件。
