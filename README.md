系统架构

```
KCT手表 (CC30设备)
     ↓ (BLE蓝牙)
BLEDebug.exe (Windows工具)
     ↓ (BLE连接)
Python采集脚本 (unified_ble_mqtt_tool.py)
     ↓ (MQTT协议)
MQTT服务器 (localhost:1883)
     ↓ (MQTT订阅)
后端服务器 (Node.js + Express)
     ↓ (WebSocket)
前端界面 (Vue.js + ECharts)
     ↓ (MySQL存储)
数据库 (健康数据持久化)
```


快速启动指南

### 步骤1：安装必要的Python包
```bash
cd Sport_Monitor\BLE
pip install paho-mqtt bleak
```

### 步骤2：启动系统服务（按顺序）

#### 终端1：启动MQTT服务器
```bash
cd Sport_Monitor
node simple_mqtt_broker.js
```
**预期输出**:
```
MQTT代理服务器运行在端口 1883
等待客户端连接...
```

#### 终端2：启动后端服务器
```bash
cd Sport_Monitor\server
npm start
```

#### 终端3：启动前端服务器
```bash
cd Sport_Monitor\client
npm run dev
```

#### 终端4：运行BLE数据采集
```bash
cd Sport_Monitor\BLE
运行心率采集代码multi_heart_rate_monitor.py
```

三、各模块详细功能设计
1. 仪表盘（系统首页）
核心作用：一眼掌握全局状态
数据概览卡片
总用户数、日活用户、月活用户
在线设备数、离线设备数、异常设备数
今日数据上报量、告警总数
健康趋势统计
近 7 天心率异常 / 睡眠异常趋势图
运动类型分布（跑步 / 骑行 / 游泳等）
设备状态监控
在线 / 离线 / 故障设备饼图
待处理告警（紧急置顶）
2. 用户管理
用户列表
搜索：手机号、昵称、ID、设备 MAC
展示：头像、账号、绑定设备数、注册时间、状态（正常 / 封禁）
操作：查看详情、编辑、禁用、解绑设备、导出数据

基本信息、账号状态
绑定设备列表
健康数据概览
操作日志
用户分组 / 标签
按年龄、性别、健康状态分组
批量推送消息
3. 设备管理（核心模块）
设备列表
字段：MAC 地址、设备型号、固件版本、绑定用户、在线状态、激活时间
搜索：MAC、型号、状态、用户
操作：查看、配置、重启、解绑、升级固件
设备状态监控
实时在线状态
信号强度、电量、定位状态
设备配置中心
心率监测间隔、步数校准、久坐提醒、睡眠监测开关
告警阈值配置（心率高低值）
固件升级管理

设备黑名单（非法设备拦截）
4. 健康数据管理
实时健康数据
心率、血氧、血压、睡眠、压力
支持按用户 / 时间筛选
历史数据查询
日 / 周 / 月报表
趋势图、导出 Excel
健康异常记录
心率过高 / 过低、睡眠严重不足
数据统计分析
群体健康报告
异常率统计
5. 运动数据管理
运动类型管理
跑步、骑行、游泳、健身等

用户、时间、时长、卡路里、配速、轨迹

运动强度分析
数据导出与报表
6. 警告管理

警告类型：心率异常、跌倒、设备离线、低电量
状态：未处理 / 已处理

处理人、处理时间、备注

新增 / 编辑 / 删除管理员
