# 心率显示前端功能修改记录

## 概述

本次修复解决了学生系统前端心率实时显示功能的关键问题，确保设备心率数据能够从APK通过MQTT服务器、后端，最终实时显示在前端页面上。

## 修改的文件列表

### 1. `Sport_Monitor/client/src/utils/request.js`
**问题**：API路径重复，请求发送到`http://localhost:5173/api/api/device-data/user/realtime`（重复了`/api`）

**修改内容**：
```javascript
// 修改前
const request = axios.create({
  baseURL: '/api', // 使用Vite代理，不需要完整URL
  timeout: 10000
})

// 修改后
const request = axios.create({
  baseURL: '', // 使用相对路径，Vite代理会处理
  timeout: 10000
})
```

**修复效果**：
- API请求路径正确：`/api/device-data/user/realtime`
- Vite代理正常工作：将请求转发到`http://localhost:3001/api/device-data/user/realtime`
- 不再出现404错误

### 2. `Sport_Monitor/client/src/pages/student/Dashboard.vue`
**问题1**：用户设备ID为undefined，无法匹配WebSocket数据
**问题2**：用户信息对象为空

**修改内容**：
```javascript
// 修改前（第65行）
userInfo: JSON.parse(localStorage.getItem('userInfo')) || {},

// 修改后
userInfo: JSON.parse(localStorage.getItem('user')) || {},

// 修改前（第362行）
const currentUser = JSON.parse(localStorage.getItem('userInfo')) || {};

// 修改后
const currentUser = JSON.parse(localStorage.getItem('user')) || {};
```

**修复效果**：
- 用户设备ID正确读取：`当前用户设备ID: device001`
- 用户信息完整加载：包含id、username、name、device_id等字段
- WebSocket数据正确匹配：`设备ID匹配，更新健康数据`

### 3. `Sport_Monitor/client/src/pages/student/HealthData.vue`
**问题**：echarts图表组件错误，页面无法正常显示

**修改内容**：
1. **移除有问题的图表组件**：
   - 删除了`LineChart`和`BarChart`组件的导入和使用
   - 移除了相关的图表数据初始化代码

2. **简化页面结构**：
   - 保留核心健康数据展示功能
   - 移除复杂的图表可视化部分
   - 专注于实时心率数据显示

3. **修复CSS样式**：
   - 修复了被截断的CSS样式
   - 确保页面布局正常

**修复效果**：
- 页面正常加载，无JavaScript错误
- 实时心率数据显示正常
- 运动状况评估功能正常
- 个性化建议功能正常

## 修复的核心问题

### 1. 数据流问题
**原始问题**：心率数据无法从前端正确显示
**修复方案**：确保数据流完整：设备 → APK → MQTT服务器 → 后端 → WebSocket → 前端

### 2. API通信问题
**原始问题**：API请求路径错误，返回404
**修复方案**：修正axios配置，确保Vite代理正常工作

### 3. 用户身份识别问题
**原始问题**：用户设备ID无法识别，WebSocket数据无法匹配
**修复方案**：统一localStorage键名，确保用户信息正确加载

### 4. 页面稳定性问题
**原始问题**：HealthData页面echarts错误导致页面崩溃
**修复方案**：移除有问题的图表组件，简化页面功能

## 技术实现细节

### 实时数据更新机制
1. **API轮询**：每10秒通过API获取最新健康数据
2. **WebSocket推送**：实时接收设备数据更新
3. **数据合并**：API数据和WebSocket数据合并显示

### 设备ID匹配逻辑
```javascript
// 在Dashboard.vue中
if (currentUser.device_id === data.deviceId) {
  console.log('设备ID匹配，更新健康数据');
  this.updateHealthDataFromWebSocket(data.data);
}
```

### 错误处理机制
1. **API失败降级**：API请求失败时使用默认值
2. **WebSocket重连**：连接断开时自动重连
3. **用户友好提示**：清晰的错误信息和连接状态显示
